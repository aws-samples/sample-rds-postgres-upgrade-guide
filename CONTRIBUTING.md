# Contributing

Thanks for your interest in contributing to the RDS PostgreSQL Upgrade Guide!

## How to Contribute

### Reporting Issues

- Open a GitHub issue for corrections, outdated information, or suggestions.
- Include the section or file name so we can find it quickly.

### Submitting Changes

1. Fork the repository.
2. Create a branch from `main` (e.g., `fix/update-extension-table`).
3. Make your changes.
4. Submit a pull request with a brief description of what you changed and why.

### What We're Looking For

- Corrections to commands, SQL queries, or AWS CLI examples
- Updates for new RDS/Aurora PostgreSQL versions or features
- Improvements to clarity or structure
- New scripts or sections that fit the guide's scope

### Style Guidelines

- Keep language concise and practical — this is a working reference, not a textbook.
- Use fenced code blocks for all SQL and bash examples.
- Link to official AWS documentation rather than duplicating content that may change.
- Add `📄 Script:` references when a corresponding script exists in `scripts/`.
